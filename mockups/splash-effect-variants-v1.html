Splash Effect Animation — Three Variants

Current system shows a flat texture at ImageScale * VirtualWidth (default ~307px) that fades in, holds, and fades out over ~550ms. These three variants explore how to add impact, motion, and weight to the same action — one image, three different arrivals.

Variant 1

Impact Burst

Image rockets from tiny to oversize, bounces back, then settles. Shockwave ring + white flash + 6 particles sell the punch.

Target
~700ms total | bounce ease-out
Variant 2

Slash Sweep

Image enters from off-target with rotation, decelerating into center. Ghost trail follows behind; a bright cut-line flashes on arrival.

Target
~660ms total | cubic-bezier decel
Variant 3

Fracture Impact

Image snaps in instantly. Six crack-lines radiate outward while the image trembles. A brief dark vignette sells the weight.

Target
~640ms total | snap + tremble

MonoGame Implementation Notes

Trigger
All three variants fire on the same ModifyHpEvent / ApplyPassiveEvent as the current SplashEffectAnimationDisplaySystem. Add an enum SplashStyle { ImpactBurst, SlashSweep, FractureImpact } field to AnimationInstance or configure per-texture.
Texture
The existing splash textures (player-attack-splash.png, enemy-attack-splash.png, gain-aegis.png, etc.) are used as-is. Companion FX (rings, particles, cracks) use small primitives or dedicated 1x1 white pixel textures tinted in the shader.
Variant 1 — Impact Burst
Image animation
Eased scale keyframes: 0.12 → 1.28 → 0.91 → 1.04 → 0.98 → 1.0 over ~380ms. Cubic-bezier(0.22, 1, 0.36, 1) for classic bounce feel.
Ring + Flash + Particles
Expanding ring: scale 0.25→3.2, alpha 0.9→0, ~350ms.
Flash ellipse: scale 0.4→1.5, alpha 0.8→0, ~180ms.
Six particles launched on staggered delays from center, each with a unique angle vector. Ring/flash/particles all removed after expiry; no persistent state beyond the animation.
Variant 2 — Slash Sweep
Image animation
Keyframes: translateX(-190) rotate(-18°) → translateX(0) rotate(0°) over ~260ms. Cubic-bezier(0.12, 0.8, 0.3, 1) — fast entry with sharp deceleration. Scale also ramps 0.55 → 1.0 and alpha 0.35 → 1.0. Final frames include a settle bounce at ~85% keyframe.
Ghost trail + Cut line
One translucent copy trails behind at offset (~-50px), fading over ~220ms.
Bright vertical cut-line flashes with scaleY keyframe: 0.2 → 1.3 → 1.0, alpha 0.9→0, triggered at ~70% of sweep progress (~180ms). Rotated ~-25° to match slash gesture.
Variant 3 — Fracture Impact
Image animation
Instant appearance (steps(1) / snap to opacity=1). Tremble: 5-frame position jitter over ~140ms: (0,0)→(5,-3)→(-5,4)→(3,2)→(-2,-1)→(0,0) with a mild scale fluctuate 1.04 → 0.98 → 1.02 → 1.0.
Crack lines + Darken vignette
Six crack lines at 60° intervals (N, NE, E, SE, SW, W) expand from center.
Each: height 0→70px, alpha 0.9→0, with translateY for radial movement.
Darken vignette: radial gradient alpha 0.55→0 over ~220ms for impact emphasis.
Shared Implementation Guidance
Companion FX approach
Use a single SpriteBatch draw with a 1×1 white pixel Texture2D. Rings: stretch the pixel into a thin rectangle, then draw N times at different rotation angles (or use a circle primitive helper). Particles: small squares drawn at offset positions from center computed via angle + distance * progress.
Scaling to viewport
Current splash uses ImageScale * VirtualWidth / texture.Width. Companion FX should scale off the same base: ring max radius can be splashSize * 1.8, crack length splashSize * 0.6, etc. This keeps effects proportional across resolutions.
