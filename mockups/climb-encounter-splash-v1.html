MonoGame Implementation Notes

Trigger and gating
On ClimbEncounterSlotSelect, queue the encounter in save state but defer ShowTransition until splash completes. Block column input via StateSingleton.IsActive or a dedicated splash component.
Data
Enemy id/name from ClimbEncounterSlotAction.SlotId → save slot → enemyId. Portrait uses the same enemy texture as battle. Gate origin can start at the clicked slot Transform bounds.
Gate Break timeline (~2.4s)
0-6% Dark shade fades in.
4-24% Gate clip expands from slot center (or screen center) to fullscreen.
14-32% Diagonal slash accent sweeps behind portrait.
16-48% Portrait scales in with orbit ring rotation settle.
38-52% Enemy name letters slam in staggered (~26ms each via keyframes).
36-44% "Prepare for battle" subtitle tracks in (with early letters).
44-76% Name and subtitle hold on screen together.
76-84% Portrait, name, and subtitle fade out together.
84-100% Global fade, publish ShowTransition { Scene = Battle }.
Rendering
Gate: animate scissor/clip rect from slot bounds to full viewport, or inverse of TransitionDisplaySystem parallelogram wipe. Slash: draw before portrait texture (behind enemy image); stretched 1x1 pixel quad at atan(-vh/vw), length VirtualWidth/cos(atan(vh/vw)), scaleX 0→1. Orbit ring: hollow rect with rotation lerp 45deg→0. ASCII-only draw strings.
Debug / SFX
System
ClimbEncounterSplashDisplaySystem with DebugEditable phase timings, DebugAction("Preview Gate Break").
Audio
Sting at gate open (~8%); optional low hit on portrait land (~30%). Layer with existing SfxTrack.Transition on handoff to battle wipe.
